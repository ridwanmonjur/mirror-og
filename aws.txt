  ---
  Key Architectural Patterns:

  - Service layer (PaymentService, EventMatchService, SocialService, etc.)
  - Request validation with custom Form Requests
  - Job queues for async processing (HandleEventJoinConfirm, HandleResults, etc.)
  - Event broadcasting (JoinEventSignuped, etc.)
  - Database transactions for data consistency
  - Caching (Redis) for performance
  - Real-time sync with Firebase Firestore

  This platform is a full-featured esports tournament management system with payment
  processing, real-time match tracking, social features, and comprehensive team/user
  management!
